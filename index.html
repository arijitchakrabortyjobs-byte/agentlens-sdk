The Platform

Five layers, one audit chain

AgentLens wraps your agent in a compliance envelope — intercepting, logging, evaluating, and reporting every decision without changing your agent code.

🔭

AI Observability

Full execution trace: every tool call, model invocation, latency, cost, and outcome — chained with SHA-256 tamper-evident hashes.

  • Millisecond-precision timestamps on every turn
  • Append-only NDJSON event store for SIEM export
  • Content hashed, never stored as raw text (DPDP-compliant)
🛡️

Runtime Security Guardrails

10 OWASP ASI-aligned runtime rules that evaluate agent output before it reaches the user — goal hijacking, privilege abuse, PII leakage.

  • BLOCK: hard stop regardless of score (PII, missing disclosure)
  • ESCALATE: routes to human reviewer for Tier 1 decisions
  • Analytics-first: rules fire on actual output, not caller assertions
🧠

Reasoning Traceability (Why-Trail)

Policy-execution evidence — what rule fired, which version, what evidence triggered it, what action was taken. Deterministic, verifiable, and independent of LLM chain-of-thought.

  • Per-rule structured evidence in every audit record
  • Satisfies RBI FREE-AI Rec 18 and EU AI Act Article 14
  • record_human_override(): immutably chained reviewer evidence
📊

Live Compliance Scoring & Audit Reporting

Real-time scoring against RBI/SEBI/DPDP frameworks. Board-ready executive dashboards, RBI examiner-ready JSON, and SIEM export.

  • ISO 31000 likelihood×severity weighted compliance scores
  • ModelCard registry: tier, version, kill switch status, vendor rights
  • Exportable to Big 4 auditors and RBI examiners
How It Works

Instrument once. Audit forever.

Works with LangChain, CrewAI, AutoGen, or raw Python. No agent code changes — wrap the span, get the audit chain.

01

Configure your entity

Declare your regulatory obligations — entity type, applicable frameworks, board policy reference, AI officer. A single config object drives all downstream compliance logic.

↳ RBI FREE-AI Pillar 4 · Governance
config.py
config = AgentLensConfig( entity_name="MyBank Ltd.", entity_type=EntityType.NBFC, regulatory_frameworks=[ RegulatoryFramework.RBI_FREE_AI, RegulatoryFramework.DPDP_2023, ], board_policy_ref="AI_POLICY_v1.0_BOARD_APR2026", pii_masking_enabled=True, )
02

Load policy rules

Attach RBI credit decision rules, SEBI algo trading controls, or custom policy rules. Rules specify the regulatory reference, risk tier, and action on fail (BLOCK / ESCALATE / WARN).

↳ RBI FREE-AI Rec 18 · SEBI AIML 2025
policy_engine.py
engine = PolicyEngine() engine.add_rules(RBIPolicy.credit_decision_rules()) # Rules include: # RBI_CREDIT_001 — policy doc ref (BLOCK if missing) # RBI_CREDIT_002 — explainability (BLOCK if missing) # RBI_CREDIT_003 — >₹10L human review (ESCALATE) # RBI_CREDIT_004 — PII masking (BLOCK if disabled)
03

Wrap your agent in an audit span

The tracer context manager records every model call, tool invocation, and decision. PII is SHA-256 hashed before it touches the log. The audit chain is tamper-evident — each record links to the previous SHA-256 hash.

↳ DPDP Act 2023 §8 · RBI MRM 2026
agent_run.py
tracer = AuditTracer(config=config, policy_engine=engine) with tracer.trace_agent("credit_agent") as span: span.set_risk_tier(RiskTier.HIGH) result = my_agent.invoke(user_query) span.record_decision( output=result, reasoning="CIBIL 724≥700, DSCR 4.08x≥2.5x — §4.2", context={"amount_inr": 500_000}, )
04

Export board-ready reports

ComplianceReporter generates an executive dashboard, per-framework scoring, and RBI examiner-ready JSON. Verify audit chain integrity with a single call — tamper detection built in.

↳ RBI FREE-AI Pillar 6 · Assurance
report.py
reporter = ComplianceReporter(tracer.get_log(), config) print(reporter.executive_dashboard()) # ✅ RBI FREE-AI: 72% (weighted) # ✅ DPDP Act: 57% (weighted) # 🔒 Audit chain: INTACT (16 events) audit_json = tracer.export_audit_report(format="json") # → Submit to RBI examiners or ingest into SIEM

Why-Trail

Policy evidence, not LLM reasoning

A 2025 study from Goodfire AI and Harvard showed LLMs commit to their answer in the first few tokens and generate subsequent "reasoning" as post-hoc justification. AgentLens's why-trail captures something different.

📌

Which policy rule fired

Rule ID, version, and regulatory reference — e.g. RBI_CREDIT_001 v1.0 mapping to RBI FREE-AI Rec 14.

🔬

What evidence triggered it

Structured evidence dict with the exact values that caused the rule to pass or fail — amount_inr: 1500000 vs threshold.

⚡

What action was taken

ALLOW / WARN / ESCALATE / BLOCK — encoded in the architecture, not in a score. BLOCK rules stop execution regardless of any other score.
